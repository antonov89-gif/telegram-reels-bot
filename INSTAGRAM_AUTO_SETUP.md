# Автопубликация в Instagram — настройка для твоего кейса

Ты выбрал `instagram_auto`. Есть 2 рабочих варианта:

### Вариант 1: Самый простой — instagrapi (рекомендую для старта)
Работает с обычным Instagram аккаунтом.

В .env пропиши:
```
IG_USERNAME=твой_логин_инсты
IG_PASSWORD=твой_пароль
```

Плюсы: 2 строки, сразу постит Reels.
Минусы: если Instagram попросит 2FA — нужно будет ввести код (я в коде добавил обработку).

Бот вызовет:
```python
from instagrapi import Client
cl = Client()
cl.login(IG_USERNAME, IG_PASSWORD)
cl.clip_upload(video_path, caption="🔥 ...")
```

### Вариант 2: Официальный Graph API (для бизнес-аккаунтов)
Нужен Facebook Developer App + бизнес Instagram.

В .env:
```
IG_USER_ID=178414...
IG_ACCESS_TOKEN=EAA...
```

Как получить:
1.  developers.facebook.com -> Создать App -> Business
2.  Добавить продукт Instagram Graph API
3.  Подключить Instagram бизнес-аккаунт к FB странице
4.  Сгенерить токен с правами `instagram_content_publish`

Для Reels видео должно лежать на ПУБЛИЧНОМ URL (S3, Cloudinary, твой сервер).
Поэтому в этом варианте бот сначала заливает на временный хостинг и потом публикует.

### Что я уже сделал в боте

В `publisher.py` оба варианта готовы:
- `publish_via_instagrapi()` — для тебя это основной
- `publish_to_instagram_graph_api()` — если перейдешь на бизнес

В Telegram боте после генерации видео будет кнопка `📤 Опубликовать в IG` — жмешь и улетает.

### Veo 3 — твой кейс (есть только Gemini API key)

Google пока не дает Veo 3 по обычному Gemini API ключу. Нужен Vertex AI.

Я сделал умный fallback в твоем боте:
1.  Агент 2 генерит ИДЕАЛЬНЫЙ промпт для Veo 3 (на английском, с камерой, светом, 9:16)
2.  Бот пытается сгенерить через API
3.  Если не получается (а с обычным ключом не получится) — отдает тебе промпт + кнопку
4.  Ты идешь в https://labs.google/fx/tools/flow — вставляешь промпт — получаешь видео за 40 сек
5.  Отправляешь видео ОБРАТНО боту файлом — и бот автоматом запостит его в IG по кнопке

Как только получишь доступ к Vertex AI:
- Включи Vertex AI API в GCP
- Поставь `GOOGLE_CLOUD_PROJECT` и `gcloud auth application-default login`
- Код уже заработает без изменений — начнет генерить напрямую.
